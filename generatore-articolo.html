Progetti correlati (sidebar)
Come si usa Genera il blocco <article> da incollare in blog.html dentro il div #blogList.
I dati vengono presi automaticamente da quanto hai compilato nel tab ①.
Metti sempre l'articolo più recente in cima alla lista.
Immagine card Se hai inserito un percorso immagine nel tab ①, appare nella card con classe has-image.
Formato consigliato: webp, 800×500px, max 100kb — usa Squoosh per ottimizzare.
Flusso di lavoro completo 1. Compila il tab ① Articolo con tutti i dati
2. Clic "Genera articolo" → scarica o copia il codice → salva come articolo-nome.html nella cartella del sito
3. Vai al tab ② Card Blog → clic "Genera card" → copia il blocco
4. Apri blog.html nel tuo editor → incolla la card dentro #blogList (i più recenti in cima)
5. Carica entrambi i file su GitHub → Netlify si aggiorna automaticamente
Tag HTML nel testo <p>testo</p> → paragrafo normale
<h2>titolo</h2> → titolo di sezione
<blockquote>"citazione"</blockquote> → citazione in evidenza con bordo magenta
<strong>parola</strong> → grassetto
<em>parola</em> → corsivo (usa il font serif)
<ul><li>voce</li></ul> → lista puntata
<figure><img src="img/blog/nome.webp" alt="..."><figcaption>didascalia</figcaption></figure> → immagine nel testo
Immagini blog — nomi liberi A differenza dei progetti, le immagini del blog non hanno un nome standardizzato.
Mettile tutte in img/blog/ con un nome descrittivo:
img/blog/spadoni.webp · img/blog/intervista-cna.webp
Formato: webp · Dimensioni card: 800×500px · Max 100kb · Usa Squoosh
Categorie multiple Puoi selezionare più categorie per lo stesso articolo — vengono tutte incluse sia nel file HTML sia nella card.
Il sistema di filtri in blog.html le gestisce automaticamente tramite lo script esistente.
